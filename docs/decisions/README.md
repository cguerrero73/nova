# Architecture Decision Records

This folder contains ADRs documenting significant architectural decisions.

## Format

Each ADR follows this structure:
- **Status**: Proposed, Accepted, Deprecated, Superseded
- **Context**: The situation that prompted this decision
- **Decision**: What was decided
- **Consequences**: Benefits and drawbacks

## Creating an ADR

1. Copy `docs/decisions/TEMPLATE.md`
2. Name with next sequential ID
3. Fill in all sections
4. Submit for review

See [TEMPLATE.md](TEMPLATE.md) for the full template.
